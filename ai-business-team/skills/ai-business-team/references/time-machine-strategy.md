# タイムマシン戦略リサーチ リファレンス

## 1. タイムマシン戦略とは

海外（主に米国・欧州・中国・東南アジア等）で既に成立・成長しているビジネスモデルやサービスを発見し、日本市場にローカライズして展開する戦略。ソフトバンクの孫正義氏が「タイムマシン経営」として実践したことで知られる。

**本質**: 海外で検証済みのPMF（Product-Market Fit）を日本市場に適用することで、ゼロからの仮説検証コストとリスクを大幅に削減する。

### なぜ有効か
- 海外で成功 = 課題が実在し、解決策が機能することが検証済み
- 日本と海外の「時差」（規制、文化、デジタル化速度の違い）が参入機会を生む
- 投資家にとっても「海外に成功事例あり」はリスク低減の強いシグナル

### 注意点・落とし穴
- 海外で成功したモデルがそのまま日本で通用するとは限らない（文化・商慣習・規制の違い）
- ローカライズの深さが成否を分ける（翻訳だけでは不十分）
- 海外の先行者がグローバル展開で日本に参入するリスク
- 既に別の日本企業が同じモデルを持ち込んでいる可能性

## 2. 調査対象の定義

### 2.1 調査すべき企業のプロファイル

| 優先度 | 対象 | 理由 |
|--------|------|------|
| **最高** | 海外でシード〜シリーズBの資金調達をした同領域スタートアップ | 市場性をVCが認め、まだ日本未参入の可能性が高い |
| **最高** | 海外で急成長中だが日本にほぼ認知がないサービス | タイムマシンの最大のチャンス |
| **高** | 海外で大きな市場を形成しているが日本では黎明期のカテゴリ | カテゴリごと持ち込む機会 |
| **高** | 海外のユニコーン/成功企業で、日本未展開のもの | モデルの有効性は実証済み |
| **中** | 海外から日本に参入済みだが浸透していないサービス | ローカライズ不足を突ける |
| **中** | 中国・東南アジアで先行しているモデル | 欧米偏重では見落とす |

### 2.2 調査すべき地域

| 地域 | 特徴 | 重点度 |
|------|------|--------|
| **米国** | スタートアップエコシステム最大、最も早いトレンド | 必須 |
| **欧州（UK/ドイツ/フランス/北欧）** | 規制先進、フィンテック・サステナビリティに強い | 必須 |
| **中国** | 独自のエコシステム、スーパーアプリ、OMO先進 | 必須 |
| **東南アジア（インド含む）** | 新興市場向けモデル、コスト構造が独特 | 推奨 |
| **イスラエル** | ディープテック、サイバーセキュリティ | 推奨 |
| **オーストラリア/カナダ** | 英語圏で中規模市場、日本と類似の課題あり | 任意 |

## 4. 分析フレームワーク

### 4.1 タイムマシン適合性評価マトリクス

各海外サービス/モデルについて以下を評価:

| 評価項目 | 1点（低） | 3点（中） | 5点（高） |
|---------|----------|----------|----------|
| **海外での検証度** | アイデア段階 | PMF達成・初期成長 | 大規模成長・カテゴリ確立 |
| **日本市場との課題共通性** | 課題が日本に存在しない | 類似の課題あり | 全く同じ課題が深刻に存在 |
| **日本での競合不在度** | 既に強い日本プレイヤーあり | 日本に類似サービスあるが弱い | 日本にほぼ存在しない |
| **ローカライズ容易性** | 法規制・商慣習の壁が高い | 中程度の適応が必要 | 最小限のローカライズで可能 |
| **先行者の日本参入リスク** | 既に日本参入準備中 | いずれ参入する可能性あり | 日本参入の兆候なし |
| **市場規模ギャップ** | 海外も日本も同規模 | 海外が2-5倍 | 海外が10倍以上 |

**総合スコア（6-30点）**:
- **25-30点**: 最有力のタイムマシン候補（即座に検討すべき）
- **18-24点**: 有望（ローカライズ戦略次第）
- **12-17点**: 条件付き（課題やリスクの克服が必要）
- **6-11点**: 不適合（別の方向を検討）

### 4.2 ローカライズ障壁の分析

| 障壁カテゴリ | 具体的チェック項目 |
|------------|------------------|
| **法規制** | 日本の法規制で同じモデルが運用可能か？許認可は？ |
| **商慣習** | 日本の商慣習（稟議、対面重視等）との相性は？ |
| **決済・インフラ** | 決済手段、物流、通信インフラの違いは？ |
| **文化・言語** | UI/UX、コミュニケーションスタイルの適応は？ |
| **市場構造** | B2Bの場合、企業規模分布や意思決定構造の違いは？ |
| **競合環境** | 日本独自の代替手段（紙、FAX、既存業務フロー）の根強さは？ |

### 4.3 タイムラインの推定

海外のトレンドが日本に到達するまでの「時差」を推定:

| 領域 | 典型的な時差 | 例 |
|------|-----------|-----|
| SaaS・クラウド | 2-5年 | Salesforce(米1999)→日本普及(2005頃〜) |
| フィンテック | 3-7年 | 規制の壁で時差が大きい |
| D2C・EC | 1-3年 | 消費者向けは比較的早い |
| HRテック | 2-4年 | 雇用慣行の違いで適応に時間 |
| ヘルスケア | 5-10年 | 医療規制で最も時差が大きい |
| AI/ディープテック | 1-2年 | 技術自体はグローバルに同時 |
