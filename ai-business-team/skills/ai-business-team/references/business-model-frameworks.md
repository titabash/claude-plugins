# ビジネスモデルフレームワーク リファレンス

## 1. Business Model Canvas（BMC）

Alexander Osterwalder による9つのビルディングブロック:

### 9つの要素

| # | 要素 | 質問 |
|---|------|------|
| 1 | Customer Segments（顧客セグメント） | 誰に価値を届けるか？最も重要な顧客は？ |
| 2 | Value Propositions（価値提案） | 顧客のどの課題を解決するか？何を届けるか？ |
| 3 | Channels（チャネル） | どうやって顧客にリーチし、価値を届けるか？ |
| 4 | Customer Relationships（顧客との関係） | どんな関係を構築・維持するか？ |
| 5 | Revenue Streams（収益の流れ） | 顧客は何に対していくら払うか？ |
| 6 | Key Resources（主要リソース） | 価値提案に必要な資産は？ |
| 7 | Key Activities（主要活動） | 価値提案を実現するための主要な活動は？ |
| 8 | Key Partnerships（主要パートナーシップ） | 主要なサプライヤーやパートナーは？ |
| 9 | Cost Structure（コスト構造） | 最も重要なコストは？ |

## 2. Lean Canvas

Ash Maurya によるスタートアップ向けBMC改良版:

| 要素 | 説明 |
|------|------|
| Problem（課題） | 上位3つの課題 |
| Existing Alternatives（既存の代替手段） | 顧客が現在使っている解決策 |
| Solution（ソリューション） | 各課題に対する解決策の概要 |
| Key Metrics（主要指標） | 成功を測る指標（AARRR等） |
| Unique Value Proposition（独自の価値提案） | 明確で差別化されたメッセージ |
| Unfair Advantage（競争優位性） | 容易にコピーできない優位性 |
| Channels（チャネル） | 顧客獲得経路 |
| Customer Segments（顧客セグメント） | ターゲット顧客とアーリーアダプター |
| Cost Structure（コスト構造） | 固定費・変動費 |
| Revenue Streams（収益の流れ） | 収益モデルと生涯価値 |

## 3. 収益モデルの類型

### サブスクリプション型
- **SaaS**: 月額/年額の定額課金（例: Slack, Notion）
- **定額制**: 使い放題モデル（例: Netflix）
- **段階制**: Free → Pro → Enterprise のティア構成
- **適用**: 継続利用、高い顧客生涯価値を期待できる場合

### トランザクション型
- **手数料**: 取引ごとに一定率を課金（例: Stripe 3.6%）
- **マーケットプレイス**: 売り手と買い手のマッチングで手数料（例: メルカリ 10%）
- **適用**: 取引頻度が高い、プラットフォーム型

### フリーミアム型
- **基本無料+有料機能**: 無料で広く獲得、有料でマネタイズ
- **転換率目安**: 2-5%（B2C）、5-15%（B2B）
- **適用**: ネットワーク効果が重要、ユーザー数が価値に直結

### 広告型
- **インプレッション/クリック課金**: CPM / CPC
- **適用**: 大量のユーザーベース、コンテンツプラットフォーム
- **注意**: CPMは$1-10程度、大規模トラフィックが必要

### ライセンス型
- **技術/IP ライセンス**: 特許、アルゴリズム、データの利用権
- **適用**: 独自技術を持つ場合、B2B

### その他
- **D2C（Direct to Consumer）**: 製造から販売まで自社完結
- **受託→自社プロダクト**: 受託で資金と知見を得て自社サービス化
- **API課金**: 使用量に応じた従量課金

## 4. プラットフォームビジネスパターン

### 両面市場の設計
1. **ニワトリと卵問題**: どちら側を先に獲得するか
2. **サブシディサイド**: 片方を無料/低価格にしてもう片方を課金
3. **ネットワーク効果**: 同側（SNS型）vs 交差側（マーケットプレイス型）

### プラットフォーム型の検討ポイント
- 需要側と供給側のバランス戦略
- 品質管理メカニズム（レビュー、審査）
- ディスインターミディエーション防止（プラットフォーム外取引の防止）

## 5. 日本市場特有のパターン

### 受託→プロダクト転換
- 日本のSaaS企業の多くが辿るパス
- 受託で業界知識と顧客基盤を構築
- ペインポイントを特定後にプロダクト化
- 例: freee、マネーフォワード

### 系列・業界団体の影響
- 大手企業との関係構築が重要
- 業界団体経由の導入パス
- 「実績」重視の文化（導入事例の重要性）

### 価格設定の特徴
- 月額課金への抵抗感（年額一括の需要）
- 「安さ」より「安心」の訴求
- 無料トライアル期間の重要性（1ヶ月が標準）

## 6. ビジネスモデル設計チェックリスト

設計時に確認すべき項目:

- [ ] 顧客は明確に定義されているか（B2B/B2C/B2B2C）
- [ ] 課題は十分に深刻か（Must-have vs Nice-to-have）
- [ ] 収益モデルは顧客の支払い意欲に合っているか
- [ ] 顧客獲得チャネルは特定されているか
- [ ] LTV > 3×CAC を達成できる見込みはあるか
- [ ] スケーラビリティはあるか（限界費用が低いか）
- [ ] 参入障壁/スイッチングコストはあるか
- [ ] ネットワーク効果はあるか
