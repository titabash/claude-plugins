# 競合分析フレームワーク リファレンス

## 1. 競合調査の実践方法論

### 1.1 多段階検索による網羅的な競合発見

**競合調査の最大の失敗は「見つけた範囲だけで分析する」こと。** 以下の多段階アプローチで網羅性を担保する。

#### ステップ1: キーワード展開（最低5パターン以上の検索を実施）

同じ課題・ソリューションでも、異なるキーワードで言及される。以下のパターンを全て試すこと:

| パターン | 検索クエリ例 |
|---------|------------|
| 課題ベース | `"{顧客の課題}" ソリューション` |
| 機能ベース | `"{主要機能}" ツール SaaS` |
| カテゴリベース | `"{業界カテゴリ}" ソフトウェア 比較` |
| 英語キーワード | `"{English keyword}" software startup` |
| ターゲットベース | `"{ターゲット顧客}" 向け サービス` |
| 類似表現 | 同義語・関連語を使った検索 |

#### ステップ2: カオスマップ・業界地図の活用

```
"{業界}" カオスマップ 2024 2025
"{業界}" 業界地図 landscape
"{業界}" market map
"{カテゴリ}" tech landscape
```

カオスマップが見つかった場合、そこに掲載されている企業を全てリストアップし、各社を個別に調査する。

#### ステップ3: レビューサイト・比較サイトでの発見

| サイト | 検索方法 | 特徴 |
|--------|---------|------|
| G2 | `site:g2.com "{category}"` | SaaSレビュー、機能比較 |
| Capterra | `site:capterra.com "{category}"` | 中小企業向けソフトウェア |
| ITreview | `site:itreview.jp "{カテゴリ}"` | 日本のSaaSレビュー |
| BOXIL | `site:boxil.jp "{カテゴリ}"` | 日本のSaaS比較 |
| Product Hunt | `site:producthunt.com "{keyword}"` | 新しいプロダクト |
| App Store / Google Play | `"{カテゴリ}" アプリ` | モバイルアプリ |

#### ステップ4: 資金調達情報からの発見

資金調達を受けているスタートアップは、VCが市場性を認めた証拠。

```
# 国内
"{業界}" スタートアップ 資金調達
site:prtimes.jp "{業界}" 資金調達
"{業界}" シリーズA シリーズB

# グローバル
"{industry}" startup funding series A
site:crunchbase.com "{industry}" Japan
"{industry}" venture capital investment 2024
```

#### ステップ5: 海外の先行事例の調査

日本市場にまだ参入していない海外プレイヤーは潜在的競合。

```
"{industry}" startup USA Europe
"{industry}" unicorn
"{industry}" Y Combinator
"{industry}" leading companies global
```

#### ステップ6: 大手企業の動向確認

大手テック企業やコンサル企業が同領域に参入していないか確認。

```
"{業界}" "{大手企業名}" 参入 新サービス
"{industry}" Google Amazon Microsoft Meta new product
"{業界}" 大手 参入 DX
```

### 1.2 競合の分類（4層＋潜在競合）

| 分類 | 定義 | 調査の重点 | 最低調査数 |
|------|------|----------|-----------|
| **直接競合** | 同じ課題を同じアプローチで解決 | 機能比較、価格、UX、市場シェア | **5社以上** |
| **間接競合** | 同じ課題を異なるアプローチで解決 | なぜ異なるアプローチを選んだか、それぞれの強み/弱み | **3社以上** |
| **代替手段** | 顧客が現在使っている手段（Excel、手作業、紙、既存業務フロー） | 切替コスト、慣性の強さ、不満点 | **2つ以上** |
| **隣接サービス** | 隣接領域のサービスで機能拡張により参入しうるもの | 拡張の容易さ、その企業のリソース | **2社以上** |
| **潜在的競合** | 将来参入しうるプレイヤー（大手テック、異業種、海外） | 参入の可能性とインパクト | **3社以上** |

### 1.3 個別競合の深掘り調査項目

各競合（直接競合・間接競合）について、以下を可能な限り調査する:

#### 基本情報
- 企業名、設立年、所在地
- 創業者の経歴・バックグラウンド
- 従業員数（可能であれば）
- 資金調達の状況（ラウンド、金額、主要投資家）

#### プロダクト分析
- **主要機能一覧**: 何ができるか
- **価格体系**: 無料プラン有無、価格帯、課金モデル（サブスク/従量/買い切り）
- **ターゲット顧客**: 誰向けか（企業規模、業種、役職）
- **技術スタック**: 使用技術（可能な範囲で）
- **UI/UXの特徴**: ユーザー体験の差別化ポイント
- **API/連携**: 他サービスとの連携、エコシステム

#### 市場ポジション
- **推定ユーザー数/顧客数**
- **推定売上/ARR**（公開情報がある場合）
- **成長速度**（従業員増、プレスリリース頻度、プロダクト更新頻度で推測）
- **主要顧客事例**（導入事例として公開されているもの）

#### 評判・ユーザーの声
```
"{競合名}" レビュー 評判 口コミ
"{競合名}" メリット デメリット
site:g2.com "{competitor name}" OR site:itreview.jp "{競合名}"
"{競合名}" から乗り換え OR "{競合名}" やめた
```

#### 戦略・方向性
- 最近のプレスリリース、ブログ記事
- 新機能のリリース傾向
- 採用情報から推測する注力領域

### 1.5 価格分析

競合の価格戦略を分析し、市場の価格帯を把握する:

| 項目 | 調査内容 |
|------|---------|
| 価格レンジ | 市場全体の最安〜最高価格 |
| 価格帯の分布 | エントリー/ミドル/エンタープライズ |
| 課金モデル | サブスク/従量/フリーミアム/買い切り |
| 無料プランの有無 | 無料枠の範囲、制限事項 |
| 価格の妥当性 | 機能対比でのコストパフォーマンス |
| 価格変更の履歴 | 値上げ/値下げのトレンド |

## 2. Porter's Five Forces（5つの競争要因）

### フレームワーク概要

| 要因 | 分析観点 | 高い場合の影響 |
|------|---------|--------------|
| 業界内の競争 | 競合の数、成長率、差別化 | 価格競争、利益率低下 |
| 新規参入の脅威 | 参入障壁の高さ | 市場シェア侵食リスク |
| 代替品の脅威 | 他の解決手段の存在 | 価格上限、顧客流出 |
| 買い手の交渉力 | 顧客の集中度、スイッチングコスト | 値引き圧力 |
| 売り手の交渉力 | サプライヤーの集中度 | コスト増加 |

### 各要因のスコアリング（1-5）

**業界内の競争**:
- 1（低）: 寡占市場、差別化大
- 3（中）: 適度な競合数、成長市場
- 5（高）: 多数の競合、コモディティ化

**新規参入の脅威**:
- 1（低）: 高い資本要件、規制障壁、ネットワーク効果
- 3（中）: 中程度の障壁
- 5（高）: 低い参入障壁、デジタル化で容易

**代替品の脅威**:
- 1（低）: 代替手段なし、高いスイッチングコスト
- 3（中）: 一部代替可能
- 5（高）: 多数の代替品、低いスイッチングコスト

**買い手の交渉力**:
- 1（低）: 分散した顧客基盤、高いスイッチングコスト
- 3（中）: 適度な集中
- 5（高）: 少数の大口顧客、低いスイッチングコスト

**売り手の交渉力**:
- 1（低）: 多数のサプライヤー、差別化なし
- 3（中）: 適度な集中
- 5（高）: 少数の独占的サプライヤー

## 3. SWOT分析

### マトリクス構造

|  | ポジティブ | ネガティブ |
|--|----------|----------|
| 内部要因 | **Strengths**（強み） | **Weaknesses**（弱み） |
| 外部要因 | **Opportunities**（機会） | **Threats**（脅威） |

### 分析のポイント
- **Strengths**: 技術力、チーム経験、既存アセット、独自データ
- **Weaknesses**: リソース不足、知名度、経験不足、資金
- **Opportunities**: 市場成長、規制変更、技術革新、競合の弱点
- **Threats**: 大手参入、規制強化、技術陳腐化、景気後退

### クロスSWOT戦略
- **SO戦略**（強み×機会）: 積極的攻勢
- **WO戦略**（弱み×機会）: 弱点を克服して機会を活かす
- **ST戦略**（強み×脅威）: 強みで脅威に対抗
- **WT戦略**（弱み×脅威）: 撤退・縮小の判断

## 4. 競合ポジショニングマップ

### 作成手順
1. 顧客にとって最も重要な差別化軸を2つ選定
2. 各競合を2軸上にプロットする
3. 空白領域（ホワイトスペース）を特定する

### 推奨する軸の候補
- 価格（低→高） vs 機能数（少→多）
- 汎用性（特化→汎用） vs カスタマイズ性（固定→柔軟）
- ターゲット規模（SMB→エンタープライズ） vs 導入容易性（難→易）
- セルフサーブ vs ハイタッチ
- 自動化度（手動→全自動）vs 対応範囲（狭→広）

### ポジショニングマップの記述フォーマット

```markdown
### ポジショニングマップ
**X軸**: {軸名}（{左端} → {右端}）
**Y軸**: {軸名}（{下端} → {上端}）

| 象限 | 位置 | プレイヤー |
|------|------|----------|
| 左上 | {X低・Y高} | 競合A, 競合B |
| 右上 | {X高・Y高} | 競合C |
| 左下 | {X低・Y低} | 競合D, 競合E |
| 右下 | {X高・Y低} | （空白 → 参入機会） |

**自社の狙うポジション**: {象限} — 理由: ...
```

## 5. Blue Ocean / Red Ocean 評価（Kim & Mauborgne）

**重要: 「競合がいる＝レッドオーシャン」は誤り。** 本質は**Value Innovation（価値革新）**の有無。
競合が多数いても、価値曲線が収束し非顧客層が大きければBlue Ocean機会は存在する。

### Red Ocean vs Blue Ocean の正しい定義

| 観点 | Red Ocean | Blue Ocean |
|------|-----------|------------|
| 市場空間 | 既存市場内で競争 | 新しい市場空間を創造 |
| 競争 | 競合に勝つ | 競争を無意味にする |
| 需要 | 既存需要を奪い合う | 新しい需要を創造・獲得 |
| 価値とコスト | 差別化かコスト削減かの二択 | 差別化とコスト削減を**同時に**実現 |

### Strategy Canvas（戦略キャンバス）テスト
1. 業界の競争要因を6-8個列挙（顧客が重視する属性）
2. 各プレイヤーの提供レベル（低→高）をプロットし「価値曲線」を描く
3. **判定基準**:
   - 全プレイヤーの曲線が似た形状 → Red Ocean（収束した戦略）
   - 提案する曲線が異なる形状を持つ → Blue Ocean候補
4. Blue Oceanの価値曲線は3つの性質を満たす:
   - **Focus（メリハリ）**: 全要因で勝とうとせず、少数に集中
   - **Divergence（差異）**: 競合と異なる曲線の形状
   - **Compelling Tagline（明快なメッセージ）**: 一文で戦略を説明できる

### Four Actions Framework（ERRC Grid）
**4つ全てに記入があって初めてBlue Ocean戦略と言える。** Raise/Createだけではコスト膨張、Eliminate/Reduceだけでは単なる低コスト戦略。

| アクション | 質問 | 効果 |
|-----------|------|------|
| **Eliminate**（排除） | 業界が当然視しているが実は不要な要素は？ | コスト削減の原資 |
| **Reduce**（削減） | 業界標準以下に下げても問題ない過剰要素は？ | コスト削減の原資 |
| **Raise**（増加） | 業界水準を大きく超えて高めるべき要素は？ | 差別化 |
| **Create**（創造） | 業界が提供したことのない新しい価値要素は？ | 新需要の創出 |

### 非顧客層の3層分析
Blue Oceanの最大の需要源は「まだ顧客でない人々」:

| 層 | 定義 | 規模目安 |
|----|------|---------|
| 第1層 | 仕方なく使っているが離脱寸前の人 | 市場の辺縁 |
| 第2層 | 評価した上で意識的に拒否している人 | 市場の外側 |
| 第3層 | 業界の存在すら選択肢として認識していない人 | **最大のプール** |

**判定**: 非顧客層が既存顧客より大きければ、Blue Ocean創出の可能性が高い。

### Blue Ocean機会の総合評価（BOIインデックス）
以下を順番に検証。1つでもNoなら再検討:

1. **Buyer Utility（顧客便益）**: 圧倒的な便益があるか？ → Yes/No
2. **Strategic Price（戦略的価格）**: マス市場がアクセスできる価格か？ → Yes/No
3. **Target Cost（目標コスト）**: その価格で利益を出せるコスト構造か？ → Yes/No
4. **Adoption（導入障壁）**: 認知・リソース・動機・組織の障壁に対策があるか？ → Yes/No

### 6つのレッドオーシャンの罠（避けるべき誤解）
1. 既存顧客の満足度向上に集中する → 非顧客層を見よ
2. ニッチ戦略と混同する → Blue Oceanはデセグメンテーション（共通点の発見）
3. 技術革新 = 市場創造と思い込む → 顧客価値に繋がらない技術は無意味
4. 差別化（プレミアム）だけでBlue Oceanと判断 → コスト削減も同時に必要
5. 低コスト戦略だけでBlue Oceanと判断 → 新価値創造も同時に必要
6. 競合の存在だけで判断する → 価値曲線の形状と非顧客層の大きさで判断せよ

## 6. 競合動向の時系列分析

単なるスナップショットではなく、競合の「動き」を捉える:

### 分析すべき時系列データ
| 項目 | データソース | 注目ポイント |
|------|------------|------------|
| 資金調達履歴 | Crunchbase, INITIAL, PR TIMES | ラウンドの間隔、金額の推移 |
| プロダクト更新 | リリースノート、ブログ | 開発速度、注力領域の変化 |
| 採用動向 | 採用ページ、Wantedly | 組織拡大の方向性 |
| パートナーシップ | プレスリリース | エコシステム戦略 |
| 価格変更 | Wayback Machine、ニュース | 価格戦略の変化 |

### 競合のフェーズ判定
| フェーズ | 兆候 | 自社への示唆 |
|---------|------|------------|
| 初期成長 | シード〜シリーズA、少人数、頻繁なピボット | 差別化で先行可能 |
| 急成長 | シリーズB以降、積極採用、マーケティング強化 | 正面衝突を避ける |
| 成熟 | IPOまたはM&A、成長鈍化、大企業化 | イノベーションのジレンマを突く |
| 衰退 | レイオフ、サービス縮小 | その顧客を獲得するチャンス |
