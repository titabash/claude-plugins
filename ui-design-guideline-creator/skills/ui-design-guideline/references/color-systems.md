# カラーシステム ベストプラクティス

## カラースケールの構築

### プライマリカラーからの展開
プライマリカラーが与えられた場合、以下の方法でカラーパレットを生成します。

#### 1. 明度スケール（9段階）
- **50**: 最も明るい（プライマリの明度を95%に）
- **100**: 非常に明るい（明度90%）
- **200**: 明るい（明度80%）
- **300**: やや明るい（明度70%）
- **400**: 少し明るい（明度60%）
- **500**: ベース（元のプライマリカラー）
- **600**: 少し暗い（明度40%）
- **700**: やや暗い（明度30%）
- **800**: 暗い（明度20%）
- **900**: 最も暗い（明度10%）

### セカンダリカラーの生成
プライマリカラーの色相を基準に：
- **補色**: 色相を180度回転
- **類似色**: 色相を30-60度回転
- **三色配色**: 色相を120度ずつ回転

### グレースケール
ニュートラルな色として、以下の9段階を用意：
```
gray-50:  #FAFAFA
gray-100: #F5F5F5
gray-200: #E5E5E5
gray-300: #D4D4D4
gray-400: #A3A3A3
gray-500: #737373
gray-600: #525252
gray-700: #404040
gray-800: #262626
gray-900: #171717
```

### セマンティックカラー
機能的な意味を持つ色：

#### Success（成功）
- ベース: `#10B981`（緑系）
- 用途: 成功メッセージ、完了状態、ポジティブなアクション

#### Warning（警告）
- ベース: `#F59E0B`（オレンジ系）
- 用途: 注意喚起、警告メッセージ

#### Error（エラー）
- ベース: `#EF4444`（赤系）
- 用途: エラーメッセージ、削除アクション、危険な操作

#### Info（情報）
- ベース: `#3B82F6`（青系）
- 用途: 情報メッセージ、ヘルプテキスト

各セマンティックカラーも50-900の明度スケールを持ちます。

## コントラスト比の検証

### WCAG 2.1 基準
- **AAレベル**
  - 通常テキスト: 4.5:1以上
  - 大きなテキスト（18pt以上または14pt太字）: 3:1以上

- **AAAレベル**
  - 通常テキスト: 7:1以上
  - 大きなテキスト: 4.5:1以上

### 推奨される組み合わせ
- 背景が明るい場合（50-200）: テキストは700-900を使用
- 背景が中間の場合（300-600）: テキストは50または900を使用
- 背景が暗い場合（700-900）: テキストは50-200を使用

## カラートークンの命名規則

### 基本構造
```
{category}-{color}-{shade}
```

例：
- `primary-blue-500`
- `semantic-success-600`
- `neutral-gray-300`

### 用途別トークン
より意味的な命名：
- `text-primary`: メインテキスト
- `text-secondary`: サブテキスト
- `text-disabled`: 無効状態のテキスト
- `bg-primary`: メイン背景
- `bg-secondary`: サブ背景
- `border-default`: デフォルトボーダー
- `border-focus`: フォーカス時のボーダー

## ダークモード対応

### カラーの反転
ダークモードでは明度スケールを反転：
- Light mode `primary-100` → Dark mode `primary-800`
- Light mode `primary-900` → Dark mode `primary-100`

### ダークモード専用の調整
- 純黒（#000000）は避け、`#0A0A0A`〜`#1A1A1A`を使用
- コントラストを若干抑えめに（WCAG AAを維持しつつ目に優しく）
- 彩度を少し下げる（10-20%程度）
